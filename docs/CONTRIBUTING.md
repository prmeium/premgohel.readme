# Contributing

Thank you for your interest in contributing to this project! The following guidelines will help you get your changes merged quickly.

## How to contribute

- Open an issue to discuss significant changes before you start coding.
- Fork the repository and create a branch named `fix/description` or `feat/description`.
- Keep changes small and focused â€” one feature or fix per PR.

## Pull request checklist

- [ ] The code builds and runs locally.
- [ ] No sensitive data or credentials are included.
- [ ] Follow the repository's coding style.
- [ ] Include a clear description of what the change does and why.

## Code style

- Use readable, consistent indentation and descriptive names.
- Keep HTML/CSS/JS simple for this static site.

## Tests

- If you add functionality that can be tested, include tests and instructions to run them.

## Review process

- PRs will be reviewed by maintainers; they may request changes.
- Rebase or merge the latest `main` before the final approval.
